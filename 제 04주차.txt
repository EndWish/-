LAN DEVICES
LAN을 구성할때 서버, 워크스테이션, 전송매체를 연결해주는 하드웨어
디피터, 허브, 브릿지, 스위치, 라우터, 게이트웨이가 있다.

Repeater(증폭기) - 잡음이나 저항으로 인해 약해진/변형된 신호를 복원해주는 하드웨어(=>전송 거리를 확장해준다.)
OSI layer 1 device
신호를 받아서 잡을을 없애고 약해진 신호를 증폭시켜주고 다음 전송매체에게 이어서 전달해준다.

Hubs(허브) - servers, workstations, printers, and other computing devices를 연결해주는 통로 역할
OSI layer 1 device
허브는 발신자와 수신자를 알려고 하지 않는다. 알 수 도 없다.
다른 "모든" 디바이스에게 정보를 전달한다(뿌려준다). = 브로드캐스트

Bridges
OSI layer 2 device
LAN(E동)을 설계할때 여러개의 작은 세그먼트(강의실)를 구성할 수 있다. 이럴때 브릿지는 각각의 세그먼트 사이에 위치하게 된다.
발신자가 위치한 세그먼트가 아닌 다른 세그먼트로 정보를 이동할 때만 브릿지가 그 데이터를 넘겨주게 된다.
반대로 같은 세그먼트에 정보를 전달한다면 그 데이터가 다른 세그먼트에 가지 못하도록 막아준다.
브릿지는 "forward(전달) if not local(세그먼트)" 이라는 단어를 적용한다.
전체 LAN의 퍼포먼스를 올리기 위해(필요없는 구간에서 데이터가 이동하는 것을 막아준다.)
각 세그먼트에 어떤 컴퓨터가 연결되어 있는지 알아야 하므로 약간의 소프트웨어가 탑재됨(MAC 식별자 이용)
MAC address table이라는 저장 공간을 가진다.
브릿지와 두 LAN 세그먼트에 연결된 컴퓨터의 MAC address 정보를 가진다. 

Switches - 브릿지와 비슷한 서비스를 제공
스위치는 직접적으로 PC나 서버, 허브, 브릿지, 다른 스위치나 라우터와 연결이 가능하다. (브릿지는 허브나 다른 브릿지와 연결됨)
목적지 발신지 구별가능해서 목적지에만 전달한다.
스위치와 브릿지의 차이점으로 스위치는 여러 포트에서 받은 데이터를 동시에 처리가능
[4-1 27:30] 그림봤을때 정보전달이 동시에 이루어 질 수 있다.

Store and Forward Switches - 데이터이동중 오류가 생겼는지 확인하는 error checking기능이 탑재된 스위치  : 은행 금전정보는 오류가 나면 안되므로 이것을 사용
오류를 확인하므로 시간이 좀더 걸리는 단점.
신뢰성이 높은 장점
오류가 확인되면 -> 발신지에 데이터 재전송을 요구한다.

Cut Through Switches - 오류체킹을 하지 않는다. : 동영상 정보같은경우 굳이 오류를 전부 확인할 필요가 없다.
Store and Forward Switches 보다는 따르지만, 전달하는 데이터에 오류가 있을 수 있다.

Higher Layer Switches - 
OSI layer 2 (MAC 주소로 전송지를 찾아 가므로 data link계층에 해당)
최근에는 스위치에 OSI layer 3 and OSI layer 4 의 기능이 부가적으로 붙는 것도 나온다.
Layer 3 switches add routing capability.
Layer 4 switches add TCP port-level services.

Routers(라우터) - 
OSI layer 3 (네트워크 계층, IP주소를 이용해 데이터를 이동시키는 장비)
데이터 패킹시 네트워크와 네트워크를 이동할 수 있도록 역할을 수행
그를 위해 라우터는 2이상의 네트워크 연결에 관여한다.
또다른 기능으로 브로드캐스트(다수의 불 특정 대상에게 데이터를 보내는 것) 데이터 패킷들을 필터링(걸러) 해주는 역할
그리고 IP주소를 이용해서 목적지까지 이동시킬 수 있는 최적의 경로를 계산해서 이동시켜줌
[그림 4-2 3:30] 네트워크가 지속적으로 확장될때 라우터가 필요하다. (라우터 회사 SYSCO )
라우터들은 라우팅 테이블이라는 정보를 가지고 있다.(라우터가 포함된 네트워크에 소속된 장치들의 IP정보, 인접한 네트워크의 장치들의 IP정보를 가지고 있다. )
라우터는 MAC대신 IP를 사용해서 목적지를 산출한다.
[그림 4-2 11:30] : 라우터는 라우터끼리 통신할 수 있는 창구가 있다.
데이터 이동상황에 따라 최단거리가 반드시 최적의 경로는 아니다.

경로를 찾는 방식:
Static Routing - 특정 네트워크에서 또다른 네트워크로 이동할때 특정 경로를 써라고 정해놓은 경우
		경로를 비교할 필요가 없다. 즉, 최적의 경로를 연산하는 시간을 아낄 수 있다.
		단점 : 새로 네트워크가 추가 됐을 때 네트워크 관리자가 지정해 주어야 한다.
		따라서 네트워크가 크게 변하지 않는 환경에서 구축하는 것이 좋다.
Dynamic routing - 발신지 IP주소와 목적지 IP주소로 최적을 경로를 매번 산출해내는 방식
		데이터를 보낼때 작은 패킷으로 나누어 보내는데, 각 패킷의 경로는 다를 수 있다.
		최적의 경로를 찾아내는 공식을 라우팅 알고리즘 이라고 한다.
		전체 네트워크에 대한 분산효과, 적절한 딜레이 없는 전송이 장점이다.
		새로 구축된 네트워크와의 통신도 어렵지 않게 가능하다.

Gateways(게이트 웨이) - 라우터와 비슷한 역할, 최근에는 중요도가 떨어짐(요즘에는 IP를 많이 쓰기 때문에)
OSI layer 3 혹은 그 이상
라우터는 양쪽의 네트워크가 IP네트워크일때 중간에서 수행하는 반면,
게이트 웨이는 양쪽의 네트워크의 언어가 다른 경우 사용한다.
[그림 4-2 26:50]

Network Backbones - 두 개의 네트워크 사이에 고속으로 데이터를 보내기 위해 하드웨어, 미디어, 프로토콜, 아키텍처를 종합적으로 언급하는 개념
[그림 4-2 31:30]
고려해야 하는 사항 2가지:
Fault tolerance(실패 참다) - 백본망에 문제가 생기더라도 지속적으로 데이터를 이동시키는 능력이 있어야 한다.
Load balancing(적제 밸러스) - 목적지까지 갈 수 있는 경로가 여럿 있어야하고, 트레픽을 분산시킬 수 있어야한다.
=> 안정적인 데이터를 보내기 위한 개념
대용량의 데이터를 전송시킬 수 있는 아키텍쳐가 구성되어야 한다.(기가 이더넷등 을 구축)

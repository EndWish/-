챕터2 - Local Area Networks(LAN) - An Introduction
LAN이 모여 좀더 큰 영역의 네트워크 - Metropolitan Area Network(MAN)
MAN이 모여서 - Wide Area Network(WAN) : 국가정도의 크기

이번주차 목차
LAN serviecs, server clients 의 개념을 사펴보자
LAN을 구성할때 서버와 클라이언트가 어떤역할을 하는지에 따라 몇가지 형태로 구분가능 => mainframe, ternimal LAN configuration
peer-to-peer(p2p), 클라이언트-도미넌트, 클라이언트/서버, 분산처리 방식 LANs(distributed processing LANs) 을 살펴보자
LAN를 구성할때 사용하는 전송매체 -> 동축케이블(coaxial), 쌍꼬임선케이블(twisted-pair), 광섬유케이블(fiber optic)의 특징을 사펴보자
무선매체에 대해 살펴보자
무선네트워크 카드에 대해 살펴보자


LAN들은 하드웨어와 소프트웨어를 이용해서 구축한다.
-워크스테이션 급이상의 컴퓨터들이 서버를 담당한다.
-운영체제가 탑재된다.
-서버컴퓨터와 LAN사이에 네트워크 인터페이스 카드가 연결고리 역할을 한다.
-LAN의 망은 유선케이블이나 무선주파수대역을 이용해 구성하게 된다.
-여러 연결장치가 필요하다( Hubs, bridges, switches, routers, and other connectivity devices )
[2주차-1,  9:00분 그림 참조]

SERVERS - 네트워크 운영체제 소프웨어가 설치되고,  LAN에 연결된 클라이언트에게 서비스를 제공하는, LAN에 연결된 컴퓨터
UP and Runnig상태가 유지되어야한다 : 전원이 들워와야하고 운영체제가 클라이언트의 요청을 처리할 준비가 되어있어야 한다.

LAN Serviecs - LAN를 구성할때 어떤 목적을 가지고 설계했냐에 따라 달라진다.
ex) data storage, printing services, centralized logon services, directory services.....
[2주차-1,  14:00분 그림 참조] : 하나의 LAN서비스에서 여러 종류의 서비스를 제공할 수도 있다.

LAN CLIENTS - LAN서비스를 제공받기 위해 LAN에 연결되어 있는 컴퓨터

Mainframes 과 Terminals : 서버가 모든 역할을 수행하고 클라이언트는 단순히 유저의 입력을 받아 메인프레임에 전달한다.
			메인프레임은 전달받은 데이터를 계산하여 결과를 터미널에 보여준다. (옛날 대형 컴퓨터 형태)
여기서 터미널은 연산능력이 없기때문에 더미터미널이라고도 부른다.
[2주차-1,  19:40분 그림 참조]

Peer-to-Peer LANs - 중앙에 큰 연산이 필요하지 않은 환경에서 조성이 된다.
		워크스테이션급 컴퓨터들을 여러대 이용해서 각각의 컴퓨터가 필요에 의해서 클라이언트나 서버로 동작할 수 있게 구성한 방식
		Peer 컴퓨터는 일반적으로 같은 workgroup으로 구성한다.
[2주차-1,  21:25분 그림 참조]

Client-dominant LANs - 대부분의 연산(데이터 조작(Data manipulation), 어플리케이션 연산)은 클라이언트에서 하고
			결과물을 LAN과 연결된 서버에 저장하는 방식
[2주차-1,  25:00분 그림 참조]

Client/Server LANs - 클라이언트와 서버가 적절한 역할분담을 통해 각자의 기능을 수행하는것
		서버는 데이터 접근과 저장을 다루고
		클라이언트에서 데이터조작을 위해 꼭필요한 정보만이 LAN을 통해 이동한다.
		서버과 클라이언트사이의 데이터 이동량을 줄여 성능을 향상시킨다.
[2주차-1,  27:00분 그림 참조]

Distributed Processing LANs - 데이터의 접근과 저장 부분이 데이터를 processing하는 부분과 별개로 떨어져 있다.
			어플리케이션 processing은 여러대의 컴퓨터(서버)에 나누어 져서 처리가 되도록 설계됨
			연산에 대한 처리요구가 빠르게 바뀔때 대응하기 좋은 방식
			예) 출시한 게임의 인기가 식었을때 서버의 일부를 떼어와 다른 게임에 이동배치가 가능하다.
[2주차-1,  29:00분 그림 참조]
		

LAN 구성
-클라이언트와 서버모드 네트워크기능이 지원되는 운영체제가 탑재되어야 한다.
-클라이언트와 서버는 하드웨어와 소프트웨어, 네트워크 인터페이스 카드를 이용해서 전송매체와 함께 LAN에 연결되도록 해야한다.

전송매체 : 크게 유선과 무선으로 나뉜다.

유선:
1) Coaxial Cable (동축케이블) :
단면을 보면 두개의 전기를 흘려보낼 수 있는 conductor(전도체)와 그것을 싸고있는 보호막으로 구성된다.
conductor중 하나는 데이터를 전기적인 신호로 이동시키는 역할
나머지 하나는 지면으로 부터 데이터를 보호하는 역할
초창기 이더넷에서 많이 사용된 케이블이다.
	1-1) Thicknet(두꺼운 동축 케이블) - 지름이 약 1cm정도 되는 케이블
		500미터 이내에 약 100대의 컴퓨터를 연결할 수 있는 케이블
		[2주차-2,  2:45분 그림 참조]
	1-2) Thinnet(얇은 동축 케이블) - 대략 0.5cm, thicknet보다 싸다 but 잡음에 버티는 정보가 약화 => 185m내 약30대 커버가능
		Thinnet Connectors : 장치,NIC,케이블 등 과  케이블을 연결하기 위해 thinnet은 BNC 커넥터 를사용한다.
		[2주차-2,  8:00분 그림 참조]

2) Twisted Pair Media (쌍꼬임선):
많이 쓰이기는 LAN installations.
두개의 구리선을 꼬아놓은 선 (잡음에 강해지기 위해서)
케이블 내에 보호막이 있냐 없냐에 따라 두가지로 나뉜다. (UTP(unshielded TP), STP(shielded TP))
케이블또한 하드웨이 이기때문에 발맞추어 발전하고 있다.
UTP Connectors : UTP 8-pin connectors(RJ-45 connector 라고도 불린다.) [2주차-2,  13:50분 그림 참조]
STP는 표피안에 EMI(Electromagnetic Interference : 전자기적인 방해)를 막아주는 금속성 재질의 보호막이 한겹더 있다. [2주차-2,  15:30분 그림 참조]
STP Connectors : UTP connector와 비슷한데 EMI보호막을 연결시켜주는 ground쉴드가 있다. (뒷면이 금속성이다.)

3) Fiber-Optic Media (광섬유 케이블):
앞선 두개의 케이블과 달리 빛이 통과할 수 있는 유리나 플라스틱을 이루어져 있다.
따라서 데이터를 빛의 형태로 주고받도록 설계됨
대용량의 데이터를 매우 빠르게 이동시키기 위한 용도
최근에는 가격이 많이 내려가서 일반 가정에서도 네트워크 구성이 가능한 상황 (가정까지 광케이블로 네트워크를 구성했을때 FTTH라고 한다.)
통상적으로 2개의 광섬유를 세트로 사용(하나는 수신용, 하나는 송신용)
구리선 기반의 케이블에 비해 관리하기가 어렵다.( 유리나 플라스틱이 쉽게 부서지기 때문 )
EMI로 부터 자유롭다.(빛을 이용하므로)
커넥터는 male connector(셋탑, tv)와 female connector(두 디바이스사이)가 있다.

무선: 전기나 광속성의 conductor(전도체)가 필요없다, 대신 transceiver나 antenna가 그역할을 대신해준다.
	Radio Frequency(라디오 주파수), infrared(인프라 레드), microwave 등의 주파수 대역을 이용
4) 라디오 주파수:
transceiver(송수신이 가능한 장비)와 antenna(안테나, 수신이 가능한 장비)가 필요하다
무선 LAN 라디오 주파수가 라디오 방송국을 방해하지는 않는다.
KCC가 라디오 주파수를 할당한다.
무선 공유기(access point)가 필요하다. [2주차-2,  29:10분 그림 참조]

5)  Infrared(적외선 통신 기법):
적은 양의 데이터를 라디오 주파수에 비해 좀 더 잛은 거리에 사용가능
잡음에도 취약한 편 : infrared는 아니지만 블루투스 같은 느낌을 떠올리면 된다.

6) Microwave:
고주파 대역이기 때문에 데이터를 멀리 보낼 수 있다.
통상적으로 건물과 건물사이를 이동시킬때 적용된다
수신할때 parablolic 안테나(접시같이 생긴 안테나)가 필요하다


Network Inerface Card
네트워크 어뎁터, 네트워크 카드, NIC등 으로 불린다.
전송매체와 네트워크 장치를 물리적으로 이어주는 역할
마더보드, 별도의 확장카드, PC카드, USE장치 와같은 형태의 네트워크 카드들이 있다. [2주차-2,  37:00분 그림 참조]
네트워크 인터페이스 카드마다 전송속도가 다르다.
이더넷, Token Ring, FDDI, ATM 등 어떤 아키텍처에 따라 거기에 맞는 인터페이스 카드가 필요하다.
역할:
송신할때 컴퓨터에서 내보내는 데이터를 전송매체의 포멧에 맞게 바꾸는 역할
수신할때는 거꾸로 전송매체에서온 데이터를 컴퓨터에 맞게 원래대로 바꾸는 역할
물리적인 node address(Mac address)를 제공한다.
데이터링크 계층에서의 역할을 수행, NIC에서 부가적인 기능을 포함하는 경우가 많다.
